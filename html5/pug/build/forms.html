Meters & Progress

The HTML <meter> element represents either a scalar value within a known range or a fractional value. By default, the meter is presented as an inline-block.

The HTML <progress> element displays an indicator showing the completion progress of a task, typically displayed as a progress bar. By default, the meter is presented as an inline-block.

Fieldset & Legend

You can use <fieldset> to group related elements in a form. By default, this tag draws a box around it's child elements. Use the <legend> to define a caption for the <fieldset> element. <legend> is not required, but can only be used within the <fieldset> and has to be it's first-child.

The <input> element is the most important form element. It can be displayed in many ways, depending on the type attribute that is given.

Select

The <select> element is used in a form to create a drop-down list. Use <option> tags inside the <select> element define the available options in the list. The HTML <optgroup> element creates a grouping of options within a <select> element. If used, then make sure to also include the 'label' attribute.

Checkboxes

<input> elements of type checkbox are rendered by default as square boxes that are checked (ticked) when activated, like you might see in an official government paper form. They allow you to select single values for submission in a form (or not).

Make sure to include a value attribute with your checkbox if you need something other than the value of 'on' to be sumbitted with the checkbox's name to the server. 'on' is the default value when the value attribute is omitted.

Radio Inputs

<input> elements of type radio are generally used in radio groups—collections of radio buttons describing a set of related options. Only one radio button in a given group can be selected at the same time. Radio buttons are typically rendered as small circles, which are filled or highlighted when selected.

Buttons

The <button> tag defines a clickable button. Inside a <button> element you can put content, like text or images. This is the difference between this element and buttons created with the <input> element.

Tip: Always specify the type attribute for a <button> element. Different browsers use different default types for the <button> element.
